# **Panel Cutting Optimization Web Application – Technical Specification**

## **1. Overview**

This application is a browser-based JavaScript web app for generating **optimized cutting patterns** for sheet
materials.
Its purpose is to arrange a set of required *Parts* onto a set of available *Stock Sheets* while minimizing waste and
cut operations.

---

## **2. High-Level Features**

* Visual web application with responsive UI layout.
* CSV import for *Parts* and *Stocks* tables.
* Manual adding the *Parts* and *Stocks*.
* Algorithm to compute **best cutting solutions** using well known algorithms used in industry.
* Interactive 3D viewport for visual drawing of cutting patterns.
* Statistics per solution and per sheet.
* Model-based architecture (Part, Stock, Settings, Solution).

## **4. Data Models / Domains**

### **4.1 Part Model**

| Field            | Type    | Description                                                                   |
|------------------|---------|-------------------------------------------------------------------------------|
| number           | number  | Number/Id of the part.                                                        |
| length           | number  | Length of the part.                                                           |
| width            | number  | Width of the part.                                                            |
| quantity         | number  | Quantity required.                                                            |
| label            | string  | Identifier or name for the part.                                              |
| enabled          | boolean | If false, part is excluded from optimization.                                 |
| ignore_direction | boolean | If true, rotation is allowed (no direction constraints).                      |
| material         | string  | One of stock material. If specified the part must be cut from selected stock. |

---

### **4.2 Stock Model**

| Field            | Type    | Description                              |
|------------------|---------|------------------------------------------|
| number           | number  | Number/Id of the stock panel.            |
| length           | number  | Length of the stock panel.               |
| width            | number  | Width of the stock panel.                |
| thickness        | number  | Thickness of the stock panel.             |
| quantity         | number  | Number of stock panels available.        |
| label            | string  | Identifier or label.                     |
| enabled          | boolean | If false, stock sheet is not used.       |
| ignore_direction | boolean | If true, rotation of stock is permitted. |
| cut_top_size     | number  | Cut restriction/offset on top edge.      |
| cut_bottom_size  | number  | Cut restriction/offset on bottom edge.   |
| cut_left_size    | number  | Cut restriction/offset on left edge.     |
| cut_right_size   | number  | Cut restriction/offset on right edge.    |

These extra “cut edges” effectively reduce usable area.

---

### **4.3 Settings Model**

| Field                       | Type   | Description                                               |
|-----------------------------|--------|-----------------------------------------------------------|
| kerf_thickness              | number | The blade/cut thickness to subtract from dimensions.      |
| default_stock_cut_perimeter | number | Global default trimming size applied to new stock panels. |

---

### **4.4 Solution Model (Generated by Algorithm)**

Each solution represents a valid arrangement of all parts on one or more stock panels.

#### **Solution Fields**

* `id`
* `used_sheets`: list of sheets with placed parts and cuts
* `unused_sheets`: list of remaining sheets
* `waste_areas`: list of areas that were cut but don't contain placed parts (waste)
* `statistics`: object containing global and per-sheet metrics

#### **Solution Statistics**

**Global**

* Total used area
* Total wasted area
* Total cuts
* Total cut length

**Per Sheet**

* Sheet used area
* Sheet waste area
* Cut count
* Total cut length
* List of placed parts
* List of cuts (coordinates/directions)

#### **Used Sheets**

* List of sheets with placed parts and cuts, each containing:
    * Sheet dimensions
    * Placed parts (with positions and orientations)
    * Cuts (with coordinates and directions)
    * Waste areas (if any)

## **6. Optimization Algorithm**

### **6.1 Goal**

Generate **best valid placemnt** of placing Parts onto available Stocks, adhering to:

* Dimensions
* Kerf thickness
* Optional rotation
* Cut perimeter restrictions
* Availability of multiple stock quantities

### **6.2 Algorithm Specification**

* Use well known algorithms such as:

    * Guillotine (binary-split) 2D packer (fast, deterministic placement)
    * Three packing heuristics / strategies:
        * minSheets — greedy-first-fit decreasing by area to minimize number of sheets used
        * minWaste — best-fit by smallest leftover area when placing a part
        * minCutLength — chooses cuts/orientations that minimize new cut length (heuristic to lower total cut length)

## **7. Drawing Requirements (3D)**

The 3D view must:

* Draw stock sheets as rectangles/parallelepiped .
* Draw parts inside sheets with:
    * labels
    * part dimensions
    * color coding (optional)

* Draw cut lines:
    * horizontal/vertical
    * showing kerf thickness
* Highlight waste regions.

## **8. User Actions**

### **8.1 CSV Upload Buttons**

* Upload CSV for parts.
* Upload CSV for stocks.

### **8.2 Optimization Button**

A “Start Optimization” button triggers:

* Validation of all inputs.
* Execution of the recursive algorithm.
* Rendering of results.

### **8.3 Stop Button**

A “Stop” button triggers:

* Stop all async processes
* Top solution is rendered on canvas

### **8.4 Solution selection**

User can select any solution from list

The canvas must render the solution.

### **8.5 Cut selection**

User can select any cut from list

The canvas must render the cut.
