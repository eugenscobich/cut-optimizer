# **Panel Cutting Optimization Web Application – Technical Specification**

## **1. Overview**

This application is a browser-based JavaScript web app for generating **optimized cutting patterns** for sheet materials.
Its purpose is to arrange a set of required *Parts* onto a set of available *Stock Sheets* while minimizing waste and cut operations.
A recursive, exhaustive search algorithm evaluates all valid combinations and produces multiple alternative solutions, each with detailed statistics.

---

## **2. High-Level Features**

* Visual web application with responsive UI layout.
* CSV import for *Parts* and *Stocks* tables.
* Manual adding the *Parts* and *Stocks*.
* Algorithm to compute **all possible cutting solutions**.
* Sortable results list (multicolumn).
* Interactive canvas for visual drawing of cutting patterns.
* Statistics per solution and per sheet.
* Model-based architecture (Part, Stock, Settings, Solution).

## **4. Data Models / Domains**

### **4.1 Part Model**

| Field            | Type    | Description                                              |
| ---------------- | ------- | -------------------------------------------------------- |
| length           | number  | Length of the part.                                      |
| width            | number  | Width of the part.                                       |
| quantity         | number  | Quantity required.                                       |
| label            | string  | Identifier or name for the part.                         |
| enabled          | boolean | If false, part is excluded from optimization.            |
| ignore_direction | boolean | If true, rotation is allowed (no direction constraints). |

---

### **4.2 Stock Model**

| Field            | Type    | Description                              |
| ---------------- | ------- | ---------------------------------------- |
| length           | number  | Length of the stock panel.               |
| width            | number  | Width of the stock panel.                |
| quantity         | number  | Number of stock panels available.        |
| label            | string  | Identifier or label.                     |
| enabled          | boolean | If false, stock sheet is not used.       |
| ignore_direction | boolean | If true, rotation of stock is permitted. |
| cut_top_size     | number  | Cut restriction/offset on top edge.      |
| cut_bottom_size  | number  | Cut restriction/offset on bottom edge.   |
| cut_left_size    | number  | Cut restriction/offset on left edge.     |
| cut_right_size   | number  | Cut restriction/offset on right edge.    |

These extra “cut edges” effectively reduce usable area.

---

### **4.3 Settings Model**

| Field                       | Type   | Description                                               |
| --------------------------- | ------ | --------------------------------------------------------- |
| kerf_thickness              | number | The blade/cut thickness to subtract from dimensions.      |
| default_stock_cut_perimeter | number | Global default trimming size applied to new stock panels. |

---

### **4.4 Solution Model (Generated by Algorithm)**

Each solution represents a valid arrangement of all parts on one or more stock panels.

#### **Solution Fields**

* `id`
* `used_sheets`: list of sheets with placed parts and cuts
* `unused_sheets`: list of remaining sheets
* `waste_parts`: list of parts that could not be placed
* `statistics`: object containing global and per-sheet metrics

#### **Solution Statistics**

**Global**

* Total used area
* Total wasted area
* Total cuts
* Total cut length
* Number of panels used
* Number of parts placed

**Per Sheet**

* Sheet used area
* Sheet waste area
* Cut count
* Total cut length
* List of placed parts
* List of cuts (coordinates/directions)

## **6. Optimization Algorithm**

### **6.1 Goal**

Generate **all valid combinations** of placing Parts onto available Stocks, adhering to:

* Dimensions
* Kerf thickness
* Optional rotation
* Cut perimeter restrictions
* Availability of multiple stock quantities

### **6.2 Algorithm Specification**

* Algorithm is recursive.
* Starts from the first part, attempts to place onto a stock sheet.
* After each placement:

  * Compute remaining regions (sub-sheets) by cutting the stock panel (taken in consideration the Kerf thickness).
  * Check if remaining parts fit.
  * Recurse into next step.
  * If there is no place on sheet, use next one
  
The algorithm must run asynchronous and in parallel. 
The stop button must finish all processes and return current completed solutions

### **6.3 Combination Exploration**

The algorithm must:

* Examine every valid permutation of part placement.
* Explore all stock panels as starting points.
* Consider rotated variants (if ignore_direction = true).
* Return only complete or partial solutions depending on settings.

### **6.4 Cutting Pattern Rules**

* Every part placement generates a cut rectangle.
* A cut rectangle reduces available subareas into:

  * Left remainder
  * Right remainder
  * Top remainder
  * Bottom remainder
* Kerf thickness is subtracted from each cut line.

## **7. Solution Sorting Requirements**

Solutions list must support multi-column sorting by:

1. **Minimizing material waste**
2. **Minimizing cut operations**
3. **Minimizing total cut length**

Sorting may be ascending/descending and combine multiple criteria.

## **9. User Actions**

### **9.1 CSV Upload Buttons**

* Upload CSV for parts.
* Upload CSV for stocks.

### **9.2 Optimization Button**

A “Start Optimization” button triggers:

* Validation of all inputs.
* Execution of the recursive algorithm.
* Rendering of results.

### **9.3 Stop Button**

A “Stop” button triggers:

* Stop all async processes
* Top solution is rendered on canvas

### **9.4 Solution selection**

User can select any solution from list

The canvas must render the solution.

  * last edited tables
  * settings
  * last solutions

* Canvas API for drawing
* Web Workers recommended for async optimization processing
